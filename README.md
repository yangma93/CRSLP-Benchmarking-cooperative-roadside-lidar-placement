# CRSLP Dataset

CRSLP (Cooperative Roadside LiDAR Placement Benchmark) is a largeвЂ‘scale virtualвЂ‘realвЂ‘fusion dataset for systematic evaluation of roadside LiDAR placement strategies. The dataset provides continuous IoU sequences, point cloud samples, and aggregated surrogate metric results across 13 realвЂ‘world locations Г— 9 traffic configurations Г— multiple placement schemes. This document describes the structure, format, and usage of the released data.

## 1. Dataset Components

The dataset is organized into three parts:

1. **Continuous IoU sequences** вЂ“ PerвЂ‘frame IoU values and sensor parameters for every location, traffic scene, and placement configuration.
2. **Point cloud sequences** (selected configurations) вЂ“ Raw LiDAR point clouds and groundвЂ‘truth bounding boxes for a subset of configurations.
3. **Aggregated benchmark table** вЂ“ A CSV file containing mIoU and corresponding surrogate metric values for all evaluated configurations.

### 2.2 File Description

**iou_sequence.txt**
- Content: PerвЂ‘frame IoU values (one float per line).  
- Length: At least 3600 lines (в‰Ґ3600 consecutive frames).

**sensor_params.txt**
- Content: Sensor placement parameters.  
- Format: One row per LiDAR sensor, each row contains 6 numbers: `x y z roll pitch yaw`.  
  - `x, y, z` : 3D coordinates (meters).  
  - `roll, pitch, yaw` : Rotation angles (degrees).

## 3. Point Cloud Sequences (Selected Configurations)

Due to the large storage requirement of full point cloud sequences (multiple LiDARs Г— thousands of frames), we release point clouds only for a subset of configurations.

### 3.1 Availability Index

A file named `pointcloud_available_configs.txt` lists all configurations for which point cloud data is provided. Each row corresponds to one configuration and follows the format:

```
site_XX, traffic_scene_XX, N_RSL, config_XX
```

Example:
```
site_03, traffic_scene_05, 2_RSL, config_02
```

The actual point cloud data for these configurations is stored in a separate downloadable archive (`PointCloud_Data.zip`). A direct download link will be provided in the repository's release section.

### 3.2 Point Cloud Data Format

For each listed configuration, the folder contains:

```
pointcloud_data/
в”њв”Ђв”Ђ site_XX_traffic_scene_XX_NRSL_config_XX/
в”‚   в”њв”Ђв”Ђ frame_000000.bin
в”‚   в”њв”Ђв”Ђ frame_000001.bin
в”‚   в”њв”Ђв”Ђ ...
в”‚   в”њв”Ђв”Ђ bbox_000000.bin
в”‚   в”њв”Ђв”Ђ bbox_000001.bin
в”‚   в””в”Ђв”Ђ ...
```

**LiDAR point clouds** (`frame_XXXXXX.bin`)
- Binary file containing a singleвЂ‘precision float array.
- After reading the entire file as a 1D array of length `M`, reshape it into a `(M//3, 3)` matrix to obtain the `(x, y, z)` coordinates of each point.
- No intensity values are provided because simulated intensity is unreliable.

**GroundвЂ‘truth bounding boxes** (`bbox_XXXXXX.bin`)
- Binary file containing a singleвЂ‘precision float array.
- After reading, reshape into a `(N_bboxes, 9)` matrix.
- Each row corresponds to one bounding box: `[x_center, y_center, z_center, length, width, height, pitch, roll, yaw]`.

**Python reading example** (requires NumPy):
```python
import numpy as np

# Read point cloud
points = np.fromfile('frame_000000.bin', dtype=np.float32)
points = points.reshape(-1, 3)  # (N_points, 3)

# Read bounding boxes
bboxes = np.fromfile('bbox_000000.bin', dtype=np.float32)
bboxes = bboxes.reshape(-1, 9)  # (N_bboxes, 9)
```

## 4. Aggregated Benchmark Table (mIoU & Surrogate Metrics)

A CSV file named `benchmark_results.csv` provides the mIoU and computed surrogate metric values for every evaluated configuration. The file contains the following columns:

| Column | Description |
|--------|-------------|
| `site` | Location index (1вЂ“13) |
| `traffic_scene` | Traffic configuration index (1вЂ“9) |
| `n_rsl` | Number of LiDARs (1, 2, 3) |
| `config_id` | Placement configuration index within that (site, traffic_scene, n_rsl) |
| `mIoU` | Mean IoU over the full sequence |
| `InfraD` | PointвЂ‘cloud density surrogate |
| `InfraNUC` | PointвЂ‘cloud nonвЂ‘uniformity surrogate |
| `cov_geo` | Geometric coverage surrogate |
| `cov_lidar` | LiDARвЂ‘aware coverage surrogate |
| `recall_rate` | Target recall surrogate |
| `PE_VGOP` | ProjectionвЂ‘based visibility surrogate |
| `S_MIG` | Shannon mutual information surrogate |
| `CE` | CrossвЂ‘entropy surrogate |
| `MDG_P` | Maximum detection gain surrogate |
| `KL` | KL divergence from NeRFвЂ‘PG |

All surrogate metrics are computed according to the definitions given in the accompanying paper.
